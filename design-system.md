## 7. 交互状态和动画

### 状态设计
```
交互状态:
- 默认: 基础样式
- 悬停: 轻微提升，颜色加深
- 聚焦: 轮廓指示，可能发光
- 按下: 轻微缩小，颜色加深
- 禁用: 透明度降低，不可点击
- 加载: 旋转指示器，禁用交互
```

### 动画时长
```
动画时长:
- 极快: 100ms (微交互)
- 快速: 200ms (状态切换)
- 中等: 300ms (页面过渡)
- 慢速: 500ms (重要动画)
- 极慢: 1000ms (特殊效果)

缓动函数:
- 标准: cubic-bezier(0.4, 0, 0.2, 1)
- 进入: cubic-bezier(0, 0, 0.2, 1)
- 退出: cubic-bezier(0.4, 0, 1, 1)
- 弹跳: cubic-bezier(0.68, -0.55, 0.265, 1.55)
```

### 常用动画模式
```
1. 淡入淡出
   - 入场: opacity 0→1，时长 200ms
   - 退场: opacity 1→0，时长 200ms

2. 滑动
   - 从上滑入: translateY(-20px)→0
   - 从下滑入: translateY(20px)→0

3. 缩放
   - 放大入场: scale(0.95)→1
   - 缩小退场: scale(1)→0.95

4. 旋转
   - 加载旋转: rotate(0deg)→rotate(360deg)，无限循环
```

### AI特定动画
```
AI生成动画:
1. 思考脉冲
   - 效果: 轻微缩放脉冲
   - 时长: 1.5秒，无限循环
   - 颜色: #8B5CF6 透明度变化

2. 生成进度
   - 效果: 波浪或粒子流动
   - 方向: 从左到右
   - 速度: 根据进度调整

3. 完成效果
   - 效果: 轻微弹跳+发光
   - 时长: 500ms
   - 颜色: #10B981 发光
```

## 9. 响应式断点

### 断点定义
```
响应式断点:
- 移动端: 0-767px
- 平板端: 768-1023px
- 桌面端: 1024-1279px
- 大桌面端: 1280px+

断点应用:
- 布局变化: 在断点处重新布局
- 组件适配: 组件内部响应式
- 内容调整: 显示/隐藏内容
```

### 响应式策略
```
移动端优先:
1. 基础样式针对移动端设计
2. 使用min-width媒体查询逐步增强
3. 保持核心功能在所有设备可用

组件响应式:
- 导航: 顶部导航→侧边导航→汉堡菜单
- 表格: 横向滚动→卡片布局
- 图片: 尺寸适配→延迟加载
```

## 10. 设计令牌

### 设计变量
```
CSS自定义属性示例:
:root {
  /* 颜色 */
  --color-primary: #4F46E5;
  --color-primary-hover: #4338CA;
  
  /* 间距 */
  --spacing-1: 4px;
  --spacing-2: 8px;
  
  /* 圆角 */
  --radius-sm: 4px;
  --radius-md: 8px;
  
  /* 阴影 */
  --shadow-sm: 0 1px 2px 0 rgba(0, 0, 0, 0.05);
  
  /* 动画 */
  --transition-fast: 200ms;
  --ease-standard: cubic-bezier(0.4, 0, 0.2, 1);
}
```

### 使用规范
```
设计令牌使用原则:
1. 一致性: 始终使用设计令牌，不写硬编码值
2. 语义化: 使用有意义的令牌名称
3. 可维护: 通过修改变量值统一更新样式
4. 可扩展: 支持主题切换和定制
```
