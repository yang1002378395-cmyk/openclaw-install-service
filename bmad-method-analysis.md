# BMAD 方法分析报告

## 概述
BMAD（BMAD Method）是一个结构化的工作流和代理管理框架，用于组织和执行复杂的多代理协作任务。它采用模块化架构，支持多种工作流和代理的集成。

## 核心架构

### 1. 目录结构
```
bmad-method/
├── _bmad/
│   ├── core/                    # 核心模块
│   │   ├── agents/             # 代理定义
│   │   ├── tasks/              # 独立任务
│   │   ├── workflows/          # 工作流程
│   │   └── config.yaml         # 核心配置
│   └── _config/                # 配置清单
│       ├── agent-manifest.csv  # 代理清单
│       ├── task-manifest.csv   # 任务清单
│       ├── workflow-manifest.csv # 工作流清单
│       └── manifest.yaml       # 项目清单
└── _bmad-output/               # 输出目录
```

### 2. 核心组件

#### 2.1 代理系统 (Agents)
- **BMAD Master**: 主执行器、知识保管员和工作流协调器
- 支持多代理协作（Party Mode）
- 每个代理有明确的角色、身份、沟通风格和原则

#### 2.2 工作流系统 (Workflows)
- **Party Mode**: 多代理对话协调
- **Brainstorming**: 创意头脑风暴会话
- 采用微文件架构，每个步骤独立
- 支持状态跟踪和顺序执行

#### 2.3 任务系统 (Tasks)
- 独立可执行的小型任务
- 包括编辑审查、文档索引、文档分片等
- 支持独立执行或集成到工作流中

### 3. 配置系统
- **config.yaml**: 用户配置（用户名、语言、输出目录）
- **manifest.yaml**: 模块安装清单
- CSV清单文件：代理、任务、工作流的注册表

## 关键特性

### 1. 多代理协作 (Party Mode)
- 协调多个代理进行自然对话
- 智能代理选择基于专业领域
- 保持代理个性一致性
- 支持代理间交叉对话

### 2. 结构化工作流
- 明确的步骤顺序
- 状态跟踪和持久化
- 用户控制点（继续/退出）
- 输出文档生成

### 3. 模块化设计
- 核心模块 + 可扩展模块
- 清晰的接口定义
- 配置驱动的工作流发现

### 4. 国际化支持
- 配置驱动的沟通语言
- 文档输出语言独立配置
- 支持多种语言环境

## 工作流示例

### Party Mode 工作流
1. **步骤1**: 代理加载和初始化
2. **步骤2**: 讨论协调
3. **步骤3**: 优雅退出

### Brainstorming 工作流
1. **步骤1**: 会话设置
2. **步骤2**: 技术选择和执行
3. **步骤3**: 想法组织
4. **步骤4**: 继续或结束

## 使用方法

### 1. 启动 BMAD Master
```
/agent bmad-master
```

### 2. 可用命令
- `[MH]`: 显示菜单帮助
- `[CH]`: 与代理聊天
- `[LT]`: 列出可用任务
- `[LW]`: 列出工作流
- `[PM]`: 启动 Party Mode
- `[DA]`: 退出代理

### 3. 获取帮助
在任何时候使用 `/bmad-help` 获取下一步建议

## 技术特点

### 1. 文件架构
- **微文件架构**: 每个步骤独立文件
- **CSV清单**: 动态发现和注册
- **YAML配置**: 结构化配置管理

### 2. 状态管理
- 前端状态跟踪
- 步骤完成记录
- 会话持久化

### 3. 代理协调
- 基于角色的代理选择
- 个性融合和一致性
- 自然对话流程管理
